* Version 3.4.0
  - New method convertToWin1251 introduced. It is used to transliterate texts from Unicode to ANSI (Latin). Acceptable characters which are not transliterated are ANSI (Latin) characters (a-z, A-Z), Cyrillic characters (à-ÿ, À-ß), digits (0-9), ¹, ˆ, §. API supports only printable ANSI (Latin) and Cyrillic characters.
  - New request property automaticConvertionToWin1251 added to ParamPicking used for transliteration of some of the text fields, using convertToWin1251 method.
